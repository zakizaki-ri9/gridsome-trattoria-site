# 個人飲食店のメニューUI 設計ガイドライン

本ドキュメントは、個人経営の飲食店（イタリアン等）がWebサイトを構築する際、負担を減らしつつ効果的にメニューを魅せるためのUI/UXガイドラインである。

## 1. 目的
- Webサイト更新の運用コストを下げる（自社サイトで全メニューのテキストや価格を管理しない）。
- 食べログやホットペッパー等の外部ポータルへの導線を明確にし、スムーズに遷移させる。

## 2. デザインパターン

### パターンA: 外部サイト誘導型（推奨）
自社サイトには「代表的なメニュー画像」数枚と「おおよその価格帯」のみを配置し、詳細は食べログやTableCheck等の外部サイトへ委ねる。

- **構成案**:
  1. セクション見出し: Menu
  2. 説明文: 「当店のメニュー（一例）です。価格や内容は変動する場合がございます。最新のメニューは食べログ、Instagram、または当店にてご確認ください。」
  3. ギャラリー: 高画質な料理写真3〜6点
  4. CTAボタン: `[食べログで最新のメニューを見る 🔗]`

### パターンB: PDF更新型
メニュー頻度が月1〜シーズンごと程度の場合、テキストで組むのではなくデザインされたPDFをアップロードし、ボタンでリンクさせる。
- **CTA**: `[今月のメニューをダウンロード (PDF)]`

### パターンC: SNS委譲型
InstagramやFacebookで「本日のメニュー」を発信している場合。
- ギャラリーの代わりにInstagramの埋め込み、もしくは「本日の黒板メニューはInstagramで発信中！」とアナウンスする。

## 3. UIの注意点とベストプラクティス
- **画像の見切れ防止**: アスペクト比を固定し、トリミングされる場合はLightboxやモーダルを用いてフルサイズで閲覧できるようにする。
- **「イメージである」ことの明記**: 提供終了したメニューが含まれる場合のクレームを防ぐため、「写真はある日の一例です」等の免責を小さく明記する。
- **外部サイト遷移の明示**: 食べログ等に飛ぶ場合は `target="_blank"` を指定し、別タブアイコン（例: ↗︎）を添えることでユーザーへ「外部に遷移する」ことを認知させる。
